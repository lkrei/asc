# Классификация архитектурных стилей 

Основная задача проекта состоит в распознавании архитектурных по изображению фасада. Данная постановка относится не просто к стандартной классификации объектов: многие стили визуально близки друг к другу, различаются деталями декора, пропорциями, ритмом оконных проемов, типами арок, колонн и кровель, а сами изображения отличаются по ракурсу, освещению и масштабу. Отдельная цель работы состоит не только в повышении качества классификации, но и в том, чтобы сделать решения моделей интерпретируемыми.

## Задача

В рамках проекта решаются четыре связанные подзадачи:

1. построить сильный baseline для классификации архитектурных стилей;
2. сравнить несколько современных архитектур глубокого обучения в режиме полного дообучения;
3. исследовать, улучшают ли качество ансамбли и гибридные подходы;
4. дополнить классификацию интерпретируемыми инструментами анализа ошибок и визуализации внимания модели.

## Сделано на текущем этапе: 

- Реализован baseline-пайплайн на `ResNet-50` и `EfficientNet-B0` в `code/baseline/`.
- Подготовлен отдельный экспериментальный контур в `code/experiments/` для полного дообучения моделей `ResNet-50`, `EfficientNet-B0`, `EfficientNet-B3`, `ViT-B/16`, `ConvNeXt-Small` и `Swin-S`.
- Реализовано усреднение вероятностей для ансамбля из пяти моделей и дополнительные пост-обработки, включая test-time augmentation.
- Построен сегментационный пайплайн в `code/segmentation/`: из масок фасадных элементов извлекаются табличные атрибуты, которые затем используются в классических ML-моделях и гибридных сетях.
- Реализованы гибридные классификаторы, объединяющие эмбеддинги изображения и фасадные признаки.
- Добавлены инструменты интерпретируемости: `Grad-CAM` для визуализации зон внимания и `SHAP` для анализа вклада табличных признаков.

## Основные результаты

Ниже приведены ключевые метрики, подтвержденные сохраненными экспериментальными артефактами.

| --- | ---: | ---: | ---: |
| Baseline `ResNet-50` | 0.6049 | 0.6065 | 0.5994 |
| Baseline `EfficientNet-B0` | 0.5854 | 0.5756 | 0.5743 |
| `ResNet-50` fine-tune | 0.7186 | 0.7097 | 0.7131 |
| `EfficientNet-B0` fine-tune | 0.6933 | 0.6833 | 0.6824 |
| `ViT-B/16` fine-tune | 0.7524 | 0.7455 | 0.7464 |
| `Swin-S` fine-tune | 0.7693 | 0.7610 | 0.7612 |
| `ConvNeXt-Small` fine-tune | **0.7804** | **0.7739** | **0.7746** |
| Ансамбль 5 моделей | **0.7934** | **0.7841** | **0.7856** |
| Hybrid follow-up `concat` | 0.7726 | 0.7642 | 0.7649 |

Краткие выводы по результатам:

- Лучшей одиночной моделью среди экспериментов с полным дообучением оказался `ConvNeXt-Small`.
- Простое усреднение вероятностей пяти моделей дало лучший итоговый результат, чем любая одиночная модель.
- `TTA` для `ConvNeXt-Small` также показал прирост качества: accuracy `0.7843`, macro F1 `0.7774`.
- Гибридная модель `concat`, объединяющая визуальные эмбеддинги и 30 фасадных атрибутов, улучшает вариант `embedding_only`, но остается немного слабее полного дообучения `ConvNeXt-Small` и ансамбля.
- Табличные модели только на сегментационных атрибутах полезны прежде всего как интерпретируемый источник признаков, а не как замена сильным визуальным моделям.

## Интерпретируемость и анализ ошибок

В проекте предусмотрен отдельный слой интерпретируемого анализа:

- `Grad-CAM` используется для визуализации областей изображения, на которые опирается модель при классификации.
- `SHAP` применяется к табличным фасадным признакам для оценки вклада отдельных атрибутов.
- Сравнение `per-class` метрик позволяет выделять пары близких и трудных стилей, где модель чаще ошибается.

Таким образом, проект не ограничивается только итоговой точностью, а позволяет анализировать, почему модель принимает то или иное решение.

## Парсер и сбор свободных изображений

В ходе проекта также был разработан вспомогательный конвейер `free_image_collector/` для аккуратного пополнения датасета свободными изображениями архитектурных фасадов.

Этот блок решает задачу сбора данных в несколько этапов:

1. `collect_free_images.py` ищет изображения по стилю через `Wikimedia Commons` и `Openverse`.
2. `batch_collect_styles.py` запускает этот процесс пакетно по списку стилей.
3. `clip_filter.py` использует `CLIP` для автоматического разделения изображений на `keep`, `review` и `reject`.
4. `build_final_dataset.py` собирает итоговый датасет из подтвержденных изображений.

Что важно в этом конвейере:

- по умолчанию отбираются только лицензии, безопасные для повторного использования без `NC` и `ND`;
- сохраняются метаданные источника, автора, лицензии и ссылки на страницу изображения;
- данные проходят первичную автоматическую очистку, но не исключают ручную проверку.

Этот конвейер использовался как исследовательский инструмент для подготовки и расширения корпуса изображений, а не как отдельный прикладной сервис.
